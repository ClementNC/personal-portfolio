## Goals of Cryptography

Cryptography provides security services over untrusted channels:

- **Confidentiality** — only intended recipients can read the message
- **Integrity** — tampering is detectable
- **Authentication** — verify the identity of communicating parties
- **Non-repudiation** — sender cannot deny having sent a message

## Kerckhoffs's Principle

A cryptosystem should be secure even if everything about the system, except the key, is public knowledge.

> "The enemy knows the system." — Claude Shannon

## Symmetric Encryption

Both parties share a secret key $k$. Encryption and decryption use the same key:

$$\text{Enc}(k, m) = c \qquad \text{Dec}(k, c) = m$$

## One-Time Pad

The OTP is **perfectly secure**: given ciphertext $c$, every plaintext $m$ is equally likely.

$$c = m \oplus k \qquad m = c \oplus k$$

**Limitation:** The key must be as long as the message and used only once — impractical for most applications.

## AES (Advanced Encryption Standard)

AES is a block cipher operating on 128-bit blocks with 128/192/256-bit keys. It uses a substitution-permutation network over 10–14 rounds.
